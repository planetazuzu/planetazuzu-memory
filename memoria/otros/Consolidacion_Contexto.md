# CONSOLIDACIÓN DE CONTEXTO Y MEMORIA

## CONTEXTO GENERAL

Perfil técnico con experiencia en entorno sanitario (ambulancias) que
está desarrollando múltiples aplicaciones digitales orientadas a:

-   Gestión de ambulancias
-   Formación profesional sanitaria
-   RRHH y bolsas de trabajo
-   Herramientas offline para emergencias
-   SaaS educativos y operativos

Trabaja principalmente con React, PWA, WordPress, NocoDB, Supabase,
Docker y autoalojamiento en servidor Ubuntu.\
Existe tendencia a rediseñar arquitectura, cambiar stack y abrir nuevos
proyectos en paralelo.

------------------------------------------------------------------------

## OBJETIVOS IDENTIFICADOS

1.  Crear una app PWA para registrar servicios urgentes con
    almacenamiento en Excel local.
2.  Desarrollar una plataforma completa de formación tipo LMS
    (multi-tenant, SaaS).
3.  Crear una app integral de gestión de ambulancias.
4.  Monetizar herramientas sanitarias.
5.  Autoalojar infraestructura.
6.  Reducir dependencia de servicios externos.
7.  Construir productos escalables.
8.  Obtener ingresos recurrentes mediante SaaS.
9.  Mantener capacidad offline-first.
10. Organizar y estabilizar el ecosistema de proyectos.

------------------------------------------------------------------------

## PROBLEMAS DETECTADOS

### Técnicos

-   Multiplicidad de stacks.
-   Cambios frecuentes de base de datos.
-   Arquitecturas híbridas complejas.
-   Falta de arquitectura final definida.
-   Repetición de rediseño de esquemas.
-   Deuda técnica acumulada.

### Estructurales

-   Demasiados proyectos en paralelo.
-   Falta de proyecto prioritario único.
-   Infraestructura sobredimensionada.
-   Monetización no validada.
-   Reinicios frecuentes de desarrollo.

------------------------------------------------------------------------

## DECISIONES TOMADAS

-   Uso de PWA como estándar.
-   Prioridad al funcionamiento offline.
-   Autoalojamiento con Docker y Portainer.
-   Eliminación parcial de servicios cloud.
-   Uso de Excel como backend local en un proyecto.
-   Revisión de suscripciones y control financiero.

------------------------------------------------------------------------

## CONTRADICCIONES

1.  Simplicidad deseada vs infraestructura compleja.
2.  Monetización rápida vs rediseño continuo.
3.  Offline minimalista vs SaaS avanzado.
4.  Excel local vs ambición escalable.
5.  Reducción de herramientas vs adopción constante.

------------------------------------------------------------------------

## CONCLUSIÓN

El principal problema es la dispersión estratégica y la acumulación de
arquitectura antes de validar monetización.

La prioridad recomendada es:

1.  Congelar expansión.
2.  Elegir un único producto.
3.  Finalizarlo.
4.  Validar monetización.
5.  Escalar después.
