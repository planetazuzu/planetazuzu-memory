# 📋 Consolidación Estratégica de Proyectos y Plataforma Institucional

## 🎯 2. Objetivo de la conversación

Analizar, estructurar y consolidar toda la información estratégica
relacionada con el desarrollo de múltiples aplicaciones institucionales
y comerciales, identificando objetivos reales, bloqueos, decisiones
técnicas e infraestructura asociada.

## 💡 4. Ideas principales y conclusiones

### Proyectos activos identificados

1.  Gestión de Transporte Sanitario (GTS).
2.  Plataforma LMS institucional (TalentOs / Skillmanager).
3.  App revisión e inventario de ambulancias.
4.  App helicópteros y puntos de evacuación.
5.  HealthGo (tipo Booking sanitario).
6.  Apps de formación paralelas (AcademiaAI / CampusFlow).
7.  Athena Pocket (IA offline móvil).
8.  Proyectos WordPress educativos.

### Infraestructura técnica

-   Servidor Ubuntu (HP G1 Mini).
-   Docker gestionado desde Portainer.
-   Nginx Proxy Manager.
-   Múltiples instancias de NocoDB.
-   Supabase (evaluado).
-   React PWA + Dexie.js.
-   WordPress + GeneratePress.
-   Stripe para pagos.
-   GitHub repos públicos.

### Decisiones tomadas

-   Priorizar sincronización Dexie ⇄ NocoDB.
-   IA configurable solo por administrador general.
-   Registro con aprobación manual.
-   Base de datos por cliente para monetización.
-   Uso de Vibe Coding (Lovable, Bolt, Firebase Studio).
-   Mantener Docker en disco principal.

### Problemas reales detectados

-   Fragmentación de proyectos.
-   Bloqueo en comercialización.
-   Exceso de ideas simultáneas.
-   Dudas sobre base de datos definitiva.
-   Falta de priorización operativa.

### Contradicciones o cambios de dirección

-   Uso de Firebase → migración a base externa.
-   Supabase evaluado pero no definitivo.
-   Dos LMS similares desarrollándose en paralelo.
-   Dudas entre NocoDB u otra base propia.
